Casos e Cenários para o /Login

################################ CASOS #################################

POST - Fazer login com sucesso na ServeRest
- Dados Válidos
- Ter um usuário cadastrado
- A requisição deve voltar um status 200

POST - Tentar fazer login com usuário não cadastrado
- Dados Válidos
- Email não cadastrado
- A requisição deve voltar um status 400

POST - Tentar fazer login com dados inválidos
- Dados Inválidos
- Usuário não pode estar cadastrado
- A requisição deve voltar um status 400

################################ CENÁRIOS #################################

Cenário: POST Fazer login com Sucesso 200
    Utilizar Dados de Usuário Existente 
    POST Endpoint /Login com dados válidos
    Validar Status Code "200"


Cenário: POST Fazer login com usuário inexistente
    Criar Dados de Usuário inexistente
    POST Endpoint /Login com usuário inexistente
    Validar Status Code "400"

Cenário: POST Fazer login com dados Inválidos
    Criar Dados Inválidos de Usuário
    POST Endpoint /Login com dados Inválidos
    Validar Status Code "400"
